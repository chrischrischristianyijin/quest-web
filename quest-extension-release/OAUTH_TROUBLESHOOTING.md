# OAuth Redirect URI Mismatch Troubleshooting

## The Problem
You're getting this error:
```
Error 400: redirect_uri_mismatch
Request details: flowName=GeneralOAuthFlow
```

## Root Cause
The redirect URI in your Google Cloud Console doesn't match the one generated by Chrome's `chrome.identity.get
RedirectURL()`.

## Step-by-Step Solution

### 1. Get Your Extension ID
1. Go to `chrome://extensions/`
2. Find your Quest extension
3. Copy the Extension ID (it's a long string like `abcdefghijklmnopqrstuvwxyz`)

### 2. Calculate the Correct Redirect URI
Your redirect URI should be:
```
https://[YOUR-EXTENSION-ID].chromiumapp.org/
```

For example, if your extension ID is `abcdefghijklmnopqrstuvwxyz`, your redirect URI should be:
```
https://abcdefghijklmnopqrstuvwxyz.chromiumapp.org/
```

### 3. Update Google Cloud Console
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Navigate to "APIs & Services" > "Credentials"
3. Find your OAuth 2.0 Client ID for Chrome Extension
4. Click on it to edit
5. In the "Authorized redirect URIs" section:
   - **Remove** any existing redirect URIs
   - **Add** the correct redirect URI from step 2
6. Click "Save"

### 4. Verify the Configuration
1. Make sure there are no extra spaces or characters
2. The extension ID must match exactly
3. The format must be exactly: `https://[extension-id].chromiumapp.org/`

### 5. Wait for Changes
- Google Cloud Console changes can take 5-10 minutes to propagate
- Try the OAuth flow again after waiting

## Debug Steps

### Option 1: Use the Debug Page
1. Open `debug-redirect-uri.html` in your browser
2. Copy the displayed redirect URI
3. Use that exact URI in Google Cloud Console

### Option 2: Check Browser Console
1. Open the extension popup
2. Open browser developer tools (F12)
3. Click "Continue with Google"
4. Check the console for the logged redirect URI
5. Use that URI in Google Cloud Console

## Common Mistakes

### ❌ Wrong Format
- `http://` instead of `https://`
- Missing `.chromiumapp.org/`
- Extra characters or spaces

### ❌ Wrong Extension ID
- Using a different extension ID
- Copying the wrong ID from extensions page

### ❌ Multiple Redirect URIs
- Having multiple redirect URIs configured
- Having old/incorrect URIs still configured

### ❌ Timing Issues
- Not waiting for Google Cloud Console changes to propagate
- Testing immediately after saving changes

## Verification Checklist

- [ ] Extension ID is correct
- [ ] Redirect URI format is: `https://[extension-id].chromiumapp.org/`
- [ ] No extra spaces or characters
- [ ] Only one redirect URI is configured
- [ ] Waited 5-10 minutes after saving changes
- [ ] Cleared browser cache/cookies
- [ ] Reloaded the extension

## Still Having Issues?

1. **Double-check the extension ID** - Make sure you're using the correct one
2. **Clear browser data** - Clear cookies and cache for Google domains
3. **Try incognito mode** - Test in a fresh browser session
4. **Check for typos** - Carefully verify every character
5. **Contact support** - If all else fails, the issue might be with Google's OAuth configuration
